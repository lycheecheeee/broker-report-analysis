### 技術決策記錄 📋
- **AI模型選擇**: qwen-2.5-7b-instruct (免費) vs qwen-2.5-72b-instruct (付費)
  - 決策: 使用7B模型，成本為0，性能足夠日常使用
  - 理由: 用戶API餘額不足，免費模型性價比高
  - 未來: 可根據需求升級到付費模型或混合使用
  
- **數據庫存儲**: SQLite (開發) → PostgreSQL (生產)
  - 決策: 當前階段SQLite足夠，後續遷移
  - 理由: 開發階段輕量級，易於部署和測試
  - 未來: 用戶量增長後遷移至PostgreSQL提升性能
  
- **前端框架**: 純HTML/CSS/JS vs React
  - 決策: 當前使用原生技術，後續可重構為React
  - 理由: 快速原型驗證，降低初期複雜度
  - 未來: 功能穩定後重構為React + TypeScript提升可維護性
  
- **後端框架**: Flask vs FastAPI vs Node.js
  - 決策: 當前使用Flask，簡單易用
  - 理由: Python生態豐富，適合AI集成
  - 未來: 可考慮FastAPI (異步支持) 或 Node.js (全棧統一)
  
- **部署策略**: 本地運行 vs 雲端部署
  - 決策: 當前本地運行，後續自動化部署到Vercel
  - 理由: 保護敏感數據，降低初期成本
  - 未來: GitHub Actions + Vercel實現零觸碰部署
